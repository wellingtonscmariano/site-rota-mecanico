# Prototype Instructions

Run the local server yourself and open the preview in the in-app browser. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Durable decisions

- Copy/positioning: NÃO usar linguagem de "transformar vidas", "mudar de vida" ou apelos emocionais similares. O foco da marca é unir pessoas que compartilham a mesma paixão (motores, ferramentas, diagnóstico, tecnologia) e a vontade de construir uma carreira de verdade — "Aqui ninguém caminha sozinho".
- ELPs (páginas de curso/workshop) são data-driven: cada oferta é um objeto (ver `diagnosticWorkshop`, `redeCanWorkshop`) registrado em `coursePages` e renderizado pelo componente genérico `CoursePage`. Para criar uma nova LP: adicionar o objeto de dados (seguindo o mesmo shape, incluindo `problemTitle`, `solutionTitle`, `audienceTitle`, `note`, `practiceTitle`/`practiceText`, `teacherName`/`teacherTagline`, `finalTitle`/`finalText`/`finalButton`), incluir no array `coursePages` e adicionar o `href` no card correspondente em `courses` (home). Copy das LPs é redigida no tom da marca a partir das listas de módulos em `Cursos Rota do Mecânico/`; dados comerciais (data, local, investimento, vagas) ficam como `A confirmar` até serem enviados. Ordem de produção segue `Planejamento_ELPs_Rota_do_Mecanico.md`.
